Rest API

    RESTful APIs use standard HTTP methods to perform operations on resources:
  • GET : Retrieve a representation of a resource.
  • POST: Create a new resource.
  • PUT: Update or create a resource.
  • DELETE: Remove a resource.
    • REST encourages a uniform and consistent interface, which is achieved through constraints like:
  • Identification of Resources: Each resource is identified by a unique URI.
  • Manipulation of Resources through Representations: Clients interact with resources through their representations (commonly in JSON or XML).
  • Hypermedia as the Engine of Application State (HATEOAS): Responses provide links or hypermedia that guide the client on the next steps.



University API

  • When the application starts, it fetches university data from the JSONPlaceholder API using the "fetch" function and the "useEffect" hook in the "UniversityList" component.
  • The fetched data is stored in the "universities" state, and the loading state is set to false when the data retrieval is complete.
  • The "UniversityList" component then maps through the array of universities and displays them in a card format.
  • If there is an error during the data fetching process, an error message is logged to the console.
  • The "App" component renders the "UniversityList" component within a container, and the CSS styles provide a clean and organized layout.
  • The result is a simple React application that dynamically fetches and displays university data from an API



Booksky

  • The HTML file provides the basic structure of the project, including a button (#addicon) on the bottom to open the popup and a container where book data will be dynamically added.
  • The CSS file defines styles for the project, including styles for the popup.
  • The JavaScript file contains functions to handle opening and closing the popup. The add function adds a new book to the main window when the user clicks the "Add" button in the popup.
  • The popup includes input fields for the book-title, author name and note, as well as "Add" and "Cancel" buttons.
  • If the user clicks "Add" after entering title and author information, a new book element is created and appended to the main window. If the user clicks "Cancel" it will be hidden.
  • This simple project provides a basic example of how you can use HTML, CSS, and JavaScript to create a popup for adding books to a main window. You can further enhance this project by adding more features and refining the user interface based on your requirements
